This course assumes that you already have pre-existing knowledge of how to program using Java, seeing as it is the most widely used language for FRC programming. This should include some basic understanding of Java syntax, functions and classes.

Where to start

Robots consist of many different parts called subsystems which each perform their own role on the robot. These are things like the drive, pick-up, and vision functions. After subsystems come the smaller components that they're made of such as motors, servos and sensors which act as the basic electrical components for the robot's functions.

You may want to start right away on the whole robot, but without the proper knowledge of programing specific components, you couldn't even do 10 lines of code.

WPILib and CTRE

Thankfully, you won't have to start from scratch. WPILib and CTRE provide various prebuilt libraries that do the dirty work for you and simplify interfacing with hardware such as motors and sensors. Whenever attempting to program a component, it's important to look at the type of component you're using, and search for it in either the WPILib or CTRE phoenix documentation, depending on the component.

Practice

It takes time to learn new functions and how to code which system you'd like, you might be working with completely different functions based on the system. Coding efficiently will take time and effort and it is not an easy process, at least you have help from the FIRST robotics libraries. A starting place would be to program a system the best you can on your own. My recommendation would be to try and program a form of intake as it is in my opinion the easiest system to code.

Starting Place

The code you want to do varies heavily based on the system, this introduction will focus on how to program a basic intake system for a robot that must pick up balls similar to the 2026 Robotics Rebuild event. If you are feeling overwhelmed or a little lost, team 5870 (League of Logic) has a great crash course for the basics of coding with the FIRST robotics coding techniques.
